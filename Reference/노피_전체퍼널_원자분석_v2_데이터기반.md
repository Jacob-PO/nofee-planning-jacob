## 물리학 제1법칙 (관성의 법칙) 정의

```
F_net = F_motivation - F_friction

F_net > 0  →  전환 발생 (운동 시작)
F_net = 0  →  정체 (현상 유지)
F_net < 0  →  이탈 (반대 방향 운동)
```

**핵심 원리**: 고객은 현재 상태(관성)를 유지하려 한다. 전환을 일으키려면 동기(F_motivation)가 마찰(F_friction)을 초과해야 한다.

## 6. 예상 F_net 개선

### 6.1 현재 vs 개선 후

| 단계 | 현재 F_net | 개선 후 F_net |
|------|-----------|--------------|
| Home→Deal | +23 | +23 (유지) |
| Deal→Click (모바일) | **-16** | **+6** |
| Deal→Click (PC) | **-4** | **+13** |

### 6.2 개선 후 계산

```
개선 후 F_net(Deal→Click, 모바일):
  F_motivation = 13 + 13(신뢰 계승) = 26
  F_friction = 12 - 5(Hero CTA) - 4(단일 CTA) - 3(이탈 버튼 축소) = 0
  F_net = 26 - 0 = +26 ✅
```

```

  = 7.2% × (1 + 0.30) × (1 + 0.25) × (1 + 0.40) × (1 + 0.10)
  = 7.2% × 1.30 × 1.25 × 1.40 × 1.10
  = 7.2% × 2.5
  ≈ 18.0%
```

## 8. 결론

### 8.1 본질적 문제 한 문장 정의

> **"Home-v2에서 축적한 신뢰 에너지(F=+17)가 Deal 페이지 진입 시 100% 증발하고, 모바일에서 초기 운동량(CTA)이 0이어서 F_net이 음수(-16)가 된다."**

### 8.2 해결 원칙

1. **에너지 보존**: 신뢰 신호를 Deal 페이지로 계승
2. **힘의 집중**: 단일 CTA로 힘 분산 방지
3. **초기 운동량**: 모바일 Hero에 CTA 추가

### 8.3 물리학적 검증

```
Newton's First Law:
"물체는 외부 힘이 없으면 현재 상태를 유지한다"

현재: F_net < 0 → 이탈 (현재 상태: 미전환 유지)
개선: F_net > 0 → 전환 (새로운 상태: 견적신청)
```
