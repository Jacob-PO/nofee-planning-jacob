# 노피 딜 페이지 원자 분해 + 제품 페이지 조화 분석

## 목적
- 화해의 원자 18개 프레임을 노피 딜 페이지에 적용해 **핵심 격차**를 수치로 명확화
- 딜 페이지의 현재 원자 구조를 재분해하고 **진짜 필요한 힘**을 도출
- 제품 페이지의 강한 원자들을 딜 페이지로 **연결/승계**하는 조화 설계

---

- **핵심 결론:** 신뢰/가치/안전 원자가 거의 보이지 않는다.

## 2) 노피 전환 모듈 설계 (화해의 "분자" 적용)

### 모듈 1: 제품 핵심 (Hero)
현재:
```
[이미지] [상품명] [월 45,000원]
```
개선:
```
[이미지] [1위 배지]
[상품명]
별점 4.8 (127명 구매)  - A01
공시가 대비 23% 저렴 - A10
월 45,000원 (총 108만원)
```

### 모듈 2: 가치/혜택
현재: 없음

추가:
```
이 조건으로 구매하면
- 공시지원금 +23% 추가 할인 - A10
- 사은품: 버즈3 (12만원 상당) - A11
- 당일 견적, 1-2일 내 답변  - A12
```

### 모듈 3: 신뢰
현재: 없음

추가:
```
이 판매점 구매 후기 (47건)  - A01
만족도 94%               - A02

자주 언급된 장점 - A04, A05
- 응답이 빨라요 (23)
- 설명이 자세해요 (18)
- 혜택이 좋아요 (15)

알아두면 좋은 점
- 평일 응답이 빨라요 (8)
- 주말은 다음날 (5)
```

### 모듈 4: 안전
현재: 없음

추가:
```
- 노피 인증 판매점        - A13
- 조건 불일치 시 구매 안 해도 OK
- 계약 전 최종 조건 확인 가능
```

### 모듈 5: CTA
현재:
```
[판매점과 채팅하기] [바로 구매] - 혼란
```
개선:
```
[전화 없이 견적 받기] - A15 단일화
조건 안 맞으면 구매 안 해도 됩니다
```

## 3) 힘의 시퀀스 재설계 (노피 버전)
1. 홈/리스트: "이 조건 인기" 배지로 관심 유도 - A18
2. Hero: 가격 앵커링 + 평점으로 1차 관성 파괴 - A01, A10
3. 혜택 모듈: 추가 혜택으로 가치 확신 - A11
4. 신뢰 모듈: 구매 후기 요약으로 불안 해소 - A04, A05
5. 안전 모듈: 리스크 제거 문구 - A13
6. CTA: 즉시 행동 유도 - A15
7. 견적 완료 후: 리뷰 요청으로 피드백 루프 - A17

## 4) 우선순위 (실행 로드맵)
- **즉시 (오늘):** 듀얼 CTA -> 단일 CTA, CTA 아래 안전 문구 추가
- **이번 주:** Hero에 "N명 구매" + 가격 앵커링 문구
- **다음 주:** 판매점 후기 요약 모듈 + 안전 모듈
- **이번 달:** 혜택 모듈 + 긴급성 배지

## 5) 딜 페이지 원자 분해 (코드 기준)

### 핵심 진단
- **모바일 Hero에 CTA가 없음** -> 전환 시작점이 비어 있음
- **좋은 데이터가 하단에 숨어 있음** -> 힘이 분산됨
- **이탈 버튼이 상단에 위치** -> 힘의 방향이 전환과 반대

### 현재 보유 원자
- **조건 명시**: 통신사/가입유형/요금제/유지기간 (조건 명확화)
- **가격 분해**: 출고가/지원금/총 할인액 (하지만 하단)
- **판매점 후기**: 리뷰/이미지/후기 수 (하단)
- **고정 CTA**: 하단 고정 (하지만 듀얼)
- **연관상품**: 유사 예산/핫딜

### 숨겨진 자산
- **판매점 평점 데이터는 존재**하지만 화면에는 숨김 처리됨
- **딜 통계/비교 데이터가 계산**되지만 화면에 노출되지 않음

### 치명적 이탈 요소
- 상단 버튼이 "기기 정보" / "다른 딜 비교"로 구성됨
- 사용자가 첫 화면에서 **전환 대신 이탈**하게 된다

## 6) 제품 페이지 원자 분해

### 제품 페이지가 가진 강한 원자
- **상품 만족도 요약**: 별점, 리뷰 수, 만족도 %
- **리뷰 키워드 태그**: 좋았던 점/고려사항 (긍정/부정 요약)
- **핵심 스펙 하이라이트**: 성능/카메라/배터리 요약
- **통신사 지원금**: 가격 앵커링 역할
- **실사용 후기 리스트**: 긴 텍스트 리뷰 + 이미지
- **최저가 비교**: 딜 리스트 + 랭킹 구조

### 문제: 신뢰의 단절
제품 페이지에서 쌓은 신뢰가 딜 페이지에서 사라진다.

```
Product 페이지: 별점 4.8 | 127개 리뷰 | 94% 만족
Deal 페이지:   월 45,000원 (신뢰 정보 없음)
```

## 7) 제품 ↔ 딜 조화 설계

### 조화 원칙
- 제품 페이지의 **상품 신뢰**는 딜 페이지의 **첫 화면**으로 승계
- 딜 페이지는 **상품 신뢰 + 판매점 신뢰**를 동시에 보여줘야 함
- 가격/혜택 정보는 **Hero에 1차 노출**, 상세는 아래에서 보완

### 딜 Hero에 추가해야 할 원자
- 상품 만족도 요약(제품 리뷰 기반)
- 판매점 평점/후기 수
- 딜 랭킹/위치 (전체 딜 대비 순위)
- 가격 앵커링(할인액/정가 대비)

### 추천 배치
```
[상품명]
별점 4.8 (이 제품 127명 만족)  - 제품 신뢰
70만원 할인 - 47개 중 3위     - 가격 + 랭킹
월 45,000원 (총 108만원)

이 판매점: 별점 4.9 (47건 후기) - 판매점 신뢰
"응답 빨라요" "설명 자세해요" - 리뷰 요약

[전화 없이 견적 받기]          - 단일 CTA
조건 안 맞으면 구매 안 해도 OK  - 안전 문구
```

## 8) 진짜 노피에 필요한 것 (본질)

### 핵심 문제
- **힘을 주는 원자가 2개뿐** (CTA, 연관상품)
- **마찰을 제거하는 원자가 거의 없음**

### 핵심 해결
- **원자 18개 중 최소 10개 이상 구현**
- **모듈 단위로 순차 배치**
- 고객이 **"스스로 결론 내렸다"고 느끼게 설계**

### 최소 구현 세트 (MVP 원자)
1. A01: 평점 + 리뷰 수 (상품 + 판매점)
2. A10: 할인 앵커링 (정가 대비 절감)
3. A15: 단일 CTA
4. A13: 안전/인증 문구
5. A04/A05: 리뷰 요약 + 키워드
6. A14: 랭킹/위치

---

## 9) 첫 번째 실행 스텝
- **CTA 단일화 + 안전 문구**부터 시작
- 다음으로 **상품 신뢰 요약을 Hero로 승계**

이 두 가지만으로도 전환 시작점의 관성을 큰 폭으로 줄일 수 있다.
