## Part 1: 물리학 제1법칙 적용

### 관성의 법칙
> "정지한 물체는 외부 힘이 작용하지 않으면 정지 상태를 유지한다"

고객의 기본 상태는 **"구매하지 않음"**이다. 전환시키려면:

```
F_net = F_motivation - F_friction > 0
```

- **F_motivation**: 가치 인식 + 신뢰 + 긴급성 + 개인화
- **F_friction**: 불확실성 + 리스크 공포 + 선택 비용 + 인지 부하

## Part 6: 본질적 문제 정의

### 문제 1: 모바일 Hero에 CTA가 없다
- PC: Hero에 "전화 연락 없는 견적 신청" 버튼 있음
- Mobile: Hero에 CTA 없음, 이탈 버튼만 있음
- **77%가 모바일인데 첫 화면에서 행동 유도 불가**

### 문제 2: 좋은 정보가 숨겨져 있다

| 정보 | 현재 위치 | 문제 |
|------|----------|------|
| 할인 금액 (70만원) | ConditionSection | Hero에 없음, 스크롤 필요 |
| 판매점 평점 | sr-only | 숨겨져 있음 |
| 판매점 리뷰 수 | StoreIntroSection | 페이지 맨 아래 |

### 문제 3: 힘의 방향이 잘못됐다

```
현재 Hero의 힘 방향:
[기기 정보] → 상품 페이지로 이탈
[다른 딜 비교] → 스크롤해서 비교 (주의 분산)

있어야 할 힘 방향:
[견적 받기] → 전환
```

### 문제 4: 신뢰 정보의 단절
- Product 페이지에서 쌓은 신뢰(평점, 만족도, 키워드)가
- Deal 페이지에서 완전히 사라짐
- 사용자는 처음부터 다시 신뢰를 쌓아야 함

## 결론

### 본질적 문제 한 문장

> **노피의 본질적 문제: 좋은 데이터(평점, 할인액, 리뷰)를 갖고 있으면서 숨기고, 모바일 Hero에 CTA 없이 이탈 버튼만 있으며, Product에서 Deal로 넘어갈 때 신뢰 정보가 단절된다.**

### 해결 방향 한 문장

> **숨긴 것 꺼내고, CTA 넣고, 이탈 버튼 줄이고, Product와 Deal의 신뢰 정보를 연결한다.**

### 화해에서 배운 핵심 원칙

1. **숫자가 설득한다**: "좋아요" (X) → "127명이 만족" (O)
2. **고객이 판단한다**: "추천합니다" (X) → "94% 만족" (O)
3. **단점도 자산이다**: 숨김 (X) → "알아두면 좋은 점" (O)
4. **시간이 압박한다**: 상시 (X) → "오늘만 이 가격" (O)
5. **행동은 항상 가능하다**: 스크롤 필요 (X) → 상시 노출 CTA (O)
