# 노피 전체 퍼널 원자분석: Home-v2 → Deal → Product → 견적신청

## 물리학 제1법칙 (관성의 법칙) 적용

> **F_net = F_motivation - F_friction**
> - F_motivation: 고객이 행동하게 만드는 동기 부여력
> - F_friction: 고객의 행동을 막는 마찰력
> - F_net > 0일 때만 전환 발생

```
├─────────────────────────────────────────────────────────────────┤
│  [Home-v2] ──────────→ [Deal Page] ──────────→ [견적신청]      │

└─────────────────────────────────────────────────────────────────┘
```

### 1.2 핵심 문제: 신뢰 신호 단절 (Trust Signal Discontinuity)

```
Home-v2                    Deal Page                Product Page
┌──────────────┐          ┌──────────────┐         ┌──────────────┐
│ ★ 별점       │    →     │              │         │ ★★★★★ 별점   │
│ 신뢰높이 N층  │    ─     │   별점 숨김   │         │ 94% 만족      │
│ 리뷰 최다    │    ─     │   신뢰 약함   │         │ 키워드 분석   │
│ 구매 수 최다  │    →     │              │         │ 강점/고려사항 │
└──────────────┘          └──────────────┘         └──────────────┘
     ↓                          ↓                        ↓
  신뢰 구축               신뢰 단절!              신뢰 강화
     │                          │                        │
     └─────── 70% 사용자가 이 경로 ─────┘         (미도달)
```

**치명적 문제**: Home-v2에서 구축한 신뢰가 Deal 페이지에서 사라짐

### 3.3 치명적 문제점

#### A. 모바일 Hero에 CTA 없음 (lines 117-138 in ProductHeroSection.tsx)
```tsx

<button onClick={scrollToLowestDeal}>다른 딜 비교</button>
```

#### B. 판매점 별점이 sr-only로 숨겨짐 (StoreIntroSection.tsx:136-142)
```tsx
{store.rating > 0 && (
  <div className="sr-only">  // ← 사용자에게 안 보임!
    <meta itemProp="ratingValue" content={String(store.rating)} />
  </div>
)}
```

#### C. 듀얼 CTA로 인한 혼란 (DealDetailClient.tsx:833-852)
```tsx
<S.ApplyButtonFull onClick={() => handleApplyEstimate('bottom', 'chat')}>
  판매점과 채팅하기  // 무엇이 더 나은지 불명확
```

## 6. 데이터 기반 근거

### 6.1 전환율 비교 (GA4 분석, 2025년 12월)

| 지표 | Before (12/1-20) | After (12/27-28) | 변화 |
|------|------------------|------------------|------|
| CTA 클릭률 | 25.4% | 17.8% | -7.6%p |
| 신청 완료율 | 66.2% | 39.1% | **-27.1%p** |
| 클릭/사용자 | 2.14 | 3.91 | +1.77 (혼란 지표) |

### 6.2 문제의 본질

1. **버튼 변경 전에도 클릭률이 25.4%로 낮았음**
   - 근본 원인: 동기 부여(F_motivation) 부족

2. **듀얼 버튼 도입 후 완료율 27.1%p 급락**
   - 클릭/사용자 1.77 증가 = 혼란으로 인한 재시도
   - 마찰력(F_friction) 급증

3. **모바일 77%인데 모바일 Hero에 CTA 없음**
   - 스크롤 없이 전환 불가
   - 이탈 유도 버튼만 존재

## 7. 본질적 해결 방안

### 7.1 우선순위 1: Deal 페이지 F_motivation 강화

#### A. 모바일 Hero에 CTA 추가
```
현재: [기기 정보] [다른 딜 비교]
개선: [견적 신청하기]
      [기기 정보] [다른 딜 비교] (축소)
```

#### B. 신뢰 신호 계승 (Trust Inheritance)
```
Home-v2에서 본 정보를 Deal 페이지에서도 표시:
- ★ 4.5 (1,234개 리뷰)
- 94% 만족
- 신뢰높이 3층
- "123명이 이 딜을 구매했어요"
```

#### C. 판매점 별점 노출 (sr-only 제거)
```tsx
// Before
<div className="sr-only">...</div>

// After
<div className="flex items-center gap-1">
  <StarIcon />
  <span>4.8</span>
  <span className="text-gray-500">(89개 후기)</span>
</div>
```

### 7.2 우선순위 2: Deal 페이지 F_friction 감소

#### A. 단일 CTA로 복귀
```
현재: [판매점과 채팅하기] [바로 구매]
개선: [전화 연락 없는 견적 신청]
```

#### B. 화해 전략 차용: Progressive Disclosure
```
Step 1: "견적 신청" (단일 버튼)
Step 2: 신청 완료 후 "판매점과 채팅하기" 옵션 제시
```

### 7.3 우선순위 3: 퍼널 재설계

#### A. Product 페이지 노출 증가
```
Home-v2 DealCard 클릭 시:
- 현재: Deal 페이지 직행
- 개선: Product 페이지 → Deal 페이지
        (신뢰 구축 후 전환)
```

#### B. Deal 페이지에 Product 신뢰 신호 삽입
```
Deal 페이지 상단에:
"이 제품 평점 ★4.5 | 94% 만족 | 배터리 오래가요, 카메라 좋아요"
```

## 10. 결론: 본질적 문제와 해결 방향

### 10.1 본질적 문제

> **"고객이 Home-v2에서 구축한 신뢰가 Deal 페이지에서 증발한다"**

- Home-v2: F_net = 51 (강함)
- Deal: F_net = 10 (약함)
- Product: F_net = 52 (강함) - 하지만 대부분 미도달

### 10.2 해결 방향

1. **Deal 페이지의 F_motivation 강화**
   - 신뢰 신호 계승
   - 사회적 증거 표시

2. **Deal 페이지의 F_friction 감소**
   - 단일 CTA
   - 모바일 Hero CTA 추가

3. **퍼널 연속성 확보**
   - Product 페이지 신뢰 신호를 Deal에서도 표시
   - 또는 Product 페이지 경유 유도

### 10.3 물리학적 표현

```
현재:  F_net(Deal) = 27 - 17 = 10  ⚠️
목표:  F_net(Deal) = 45 - 5 = 40   ✅
                     ↑      ↑
                  신뢰신호   단일CTA
                   추가     + Hero CTA
```
