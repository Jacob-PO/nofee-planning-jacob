## 1. 분석 프레임워크: 물리학 제1법칙

### 관성의 법칙

> "정지한 물체는 외부 힘이 작용하지 않으면 정지 상태를 유지한다"

고객의 기본 상태는 **"구매하지 않음"**이다. 전환시키려면:

```
F_net = F_motivation - F_friction > 0
```

| 요소 | 구성 |
|------|------|
| **F_motivation** (동기) | 가치 인식 + 신뢰 + 긴급성 + 개인화 |
| **F_friction** (마찰) | 불확실성 + 리스크 공포 + 선택 비용 + 인지 부하 |

### 핵심 원리

- **F_net > 0** → 전환 발생 (운동 시작)
- **F_net = 0** → 정체 (현상 유지)
- **F_net < 0** → 이탈 (반대 방향 운동)

## 3. 본질적 문제 진단

### 3.1 문제 1: 모바일 Hero에 CTA가 없다 (12/24 변경으로 제거됨)

```tsx
// PC에는 CTA가 있음 (ProductHeroSection.tsx line 194-203)
<button id="deal-top-cta" onClick={onApplyClick}>
  전화 연락 없는 견적 신청
</button>

// 모바일에는 CTA가 없음! (line 117-138)
// 12/24 커밋 a6ca68b에서 제거됨
// 대신 이탈 버튼 2개만 있음
<div className="grid grid-cols-2 gap-2 mt-4 tablet:hidden">
  <button onClick={goToProductPage}>기기 정보</button>   // → 이탈
  <button onClick={scrollToLowestDeal}>다른 딜 비교</button> // → 스크롤
</div>
```

**84.3%가 모바일 트래픽인데 Hero에 CTA가 없다.** (GA4 실측)

#### 역설: CTA 제거가 전환율을 높인 이유

| 가설 | 설명 |
|------|------|
| **힉스의 법칙** | 선택지 감소 → 결정 속도 향상 |
| **하단 CTA 집중** | Hero에서 분산되던 클릭이 하단 CTA로 집중 |
| **스크롤 유도 성공** | "다른 딜 비교" 버튼 → 하단까지 스크롤 → CTA 노출 |

#### 여전한 문제

- First View에서 즉각 전환 기회 상실
- 스크롤이라는 추가 마찰 존재
- 모바일 84% 사용자에게 불리한 UX

### 4.2 화해의 6가지 원자 전략

#### 원자 #1: 숫자로 신뢰 구축

| 화해의 표현 | 본질 |
|------------|------|
| `4.73 (22,287)` | "2만 명이 검증했다" |
| `4점 이상 리뷰가 94%` | "대부분 만족했다" |
| `보습이 잘 돼요 137` | "137명이 같은 말을 했다" |

**원자 공식**: `[숫자] + [검증된 사실]`

#### 원자 #2: 질문하지 않고 답만 준다

| 고객의 숨은 질문 | 화해의 답변 방식 |
|-----------------|-----------------|
| "좋은 제품이야?" | `4점 이상 리뷰가 94%` |
| "뭐가 좋아?" | `좋았던 점: 보습이 잘 돼요` |
| "단점은?" | `아쉬운 점: 유분이 있어요` |
| "내 피부에 맞아?" | `건성 피부 만족도가 높은 제품` |
| "비싼 거야?" | `55% 27,900원` (원가 대비 표시) |

**원자 공식**: `[고객 질문 예측] → [즉시 답변]`

#### 원자 #3: 분류는 고객 언어로

```
좋았던 점           아쉬운 점
─────────────      ─────────────
보습이 잘 돼요 137   유분이 있어요 18
수분이 있어요 110   알러지 반응이 와요 7
흡수가 잘 돼요 101  담담해요 6
```

**관찰**:
- "보습력 우수" (X) → "보습이 잘 돼요" (O)
- "점도가 낮음" (X) → "끈적하지 않아요" (O)

**원자 공식**: `[고객이 실제로 하는 말] 그대로 사용`

#### 원자 #4: 판단은 고객 몫

화해는 **"이 제품이 좋다"고 말하지 않는다**.

```
성분 구성
● 1-2 낮은 위험  ● 3-6 중간 위험  ● 7-10 높은 위험
```

**원자 공식**: `[객관적 정보] + [판단 도구] → 고객이 스스로 결론`

#### 원자 #5: 개인화는 라벨링으로

```
건성 피부 만족도가 높은 제품

피부 타입별 성분
지성 피부 ████░░ 0개
건성 피부 ████████ 3개
민감성 피부 ██░░░░ 1개
```

**원자 공식**: `[고객 속성] ↔ [제품 속성] 매칭 시각화`

#### 원자 #6: 행동 유도는 맥락에 맞게

| 위치 | CTA | 이유 |
|-----|-----|------|
| 상단 | `장바구니 담기` `바로 구매` | 이미 결정한 고객용 |
| 리뷰 섹션 | `리뷰 305개 전체보기 >` | 더 확인하고 싶은 고객용 |
| 하단 고정 | `장바구니 담기` `바로 구매` | 스크롤 후 결정한 고객용 |

**원자 공식**: `[고객의 현재 마음 상태] → [그에 맞는 다음 행동]`

### 4.3 화해 전환 공식의 힘 합산

```
F_motivation (동기)
├─ 가격 앵커링: +60
├─ 시간 압박: +95
├─ 사회적 증거: +115
├─ 개인화: +40
├─ 랭킹 권위: +135
└─ 추가 혜택: +75
= 총 동기력: +520

F_friction (마찰)
├─ 리스크 제거: -90
├─ 정보 계층화: -75
├─ 성분 투명성: -90
├─ AI 분석: -95
├─ 듀얼 CTA: -30
├─ 하단 고정: -75
└─ 연관 추천: -30 (이탈 비용)
= 총 마찰력: -485

F_net = 520 - (-485) = +1005 → 강력한 전환 유도
```

## 10. 핵심 원칙 요약

### 10.1 화해에서 배운 5가지 전환 원칙

| 원칙 | 잘못된 방식 | 올바른 방식 |
|------|-----------|-----------|
| **1. 숫자가 설득한다** | "좋아요" | "127명이 만족" |
| **2. 고객이 판단한다** | "추천합니다" | "94% 만족" |
| **3. 단점도 자산이다** | 숨김 | "알아두면 좋은 점" |
| **4. 시간이 압박한다** | 상시 | "오늘만 이 가격" |
| **5. 행동은 항상 가능하다** | 스크롤 필요 | 상시 노출 CTA |

### 10.2 화해의 3가지 소통 원칙

1. **숫자로 말한다**
   - 주관적 표현 배제
   - 검증된 데이터만 제시
   - 예: "좋아요" (X) → "94%가 만족" (O)

2. **고객 언어를 쓴다**
   - 전문 용어 배제
   - 고객이 실제로 하는 말 사용
   - 예: "보습력 우수" (X) → "보습이 잘 돼요" (O)

3. **판단을 강요하지 않는다**
   - 정보와 도구만 제공
   - 결론은 고객이 내린다
   - 예: "추천합니다" (X) → "건성 피부 만족도 높음" (O)

### 10.3 노피 적용 변환

**Before (노피):**
```
[큰 화면] [하루 종일 OK] [고성능] [사진 잘 나옴]
```

**After (화해 방식):**
```
이 폰을 산 고객 94%가 만족했어요

자주 언급된 장점
├─ 배터리 오래가요 (47회)
├─ 화면이 커요 (32회)
└─ 카메라 좋아요 (28회)

알아두면 좋은 점
├─ 무거워요 (12회)
└─ 발열 있어요 (8회)
```

### 10.4 해결 방향 한 문장

> **숨긴 것 꺼내고, CTA 넣고, 이탈 버튼 줄이고, Product와 Deal의 신뢰 정보를 연결한다.**

### 10.5 최소 구현 세트 (MVP 원자)

1. **A01**: 평점 + 리뷰 수 (상품 + 판매점)
2. **A10**: 할인 앵커링 (정가 대비 절감)
3. **A15**: 단일 CTA
4. **A13**: 안전/인증 문구
5. **A04/A05**: 리뷰 요약 + 키워드
6. **A14**: 랭킹/위치

### 10.6 첫 번째 실행 스텝

1. **CTA 단일화 + 안전 문구**부터 시작
2. 다음으로 **상품 신뢰 요약을 Hero로 승계**

이 두 가지만으로도 전환 시작점의 관성을 큰 폭으로 줄일 수 있다.

## 결론

### 본질적 문제 한 문장

> **노피의 본질적 문제: 좋은 데이터(평점, 할인액, 리뷰)를 갖고 있으면서 숨기고, 모바일 Hero에 CTA 없이 이탈 버튼만 있으며, Product에서 Deal로 넘어갈 때 신뢰 정보가 단절된다.**

### 물리학적 검증

```
Newton's First Law:
"물체는 외부 힘이 없으면 현재 상태를 유지한다"

현재: F_net < 0 → 이탈 (현재 상태: 미전환 유지)
개선: F_net > 0 → 전환 (새로운 상태: 견적신청)
```

### 화해가 하는 일은 단 하나:

> **"다른 고객들의 경험을 정리해서 보여준다"**

화해는 절대로:
- "이 제품 좋아요" 라고 말하지 않는다
- "당신에게 맞아요" 라고 말하지 않는다
- "사세요" 라고 말하지 않는다

화해는 오직:
- "2만 명이 평가했고, 94%가 만족했다"
- "137명이 '보습이 잘 된다'고 했다"
- "건성 피부 고객의 만족도가 높다"

**판단의 주체는 항상 고객이다.**
화해는 판단에 필요한 **정보와 도구**만 제공한다.

**고객이 스스로 "사야겠다"고 결론 내리게 한다.**

이것이 마찰을 최소화하면서 동기를 극대화하는 **물리학적으로 최적화된 전환 설계**다.

## 부록 C: 전체 퍼널 물리학 법칙 매핑

### C.1 적용된 물리학 법칙

| 분야 | 법칙 | 전환 맥락 적용 |
|------|------|---------------|
| 고전역학 | 뉴턴 제1법칙 (관성) | 미전환 상태 유지 성향 |
| | 뉴턴 제2법칙 (F=ma) | 전환력 = 동기 × 가속도 |
| | 운동량 보존 | 페이지 간 관심/신뢰 유지 |
| | 충격량 (J=FΔt) | 강한 CTA를 적시에 노출 |
| 에너지 | 에너지 보존 | 신뢰 에너지의 이동/손실 |
| 열역학 | 엔트로피 증가 | 선택지 많음 → 이탈 |
| 양자역학 | 불확정성 원리 | 가격/조건 불확실성 해소 |
| | 양자 터널링 | 마찰 장벽 돌파 확률 |

### C.2 핵심 공식

```
전환 발생 조건:
F_net = F_motivation - F_friction > 0

동기력:
F_motivation = (신뢰 × 가치 × 긴급성) / 인지부하

마찰력:
F_friction = 불확실성 + 리스크공포 + 선택비용 + 인지부하

엔트로피:
S = log(n)  // 선택지 n이 많을수록 S↑ → 이탈↑
```

### C.3 전체 퍼널 F_net 현황

```
퍼널 포인트별 F_net:

회원가입:
  Welcome    →  Terms     →  WishInfo  →  Completed
  [F=+0.1]     [F=-0.5]     [F=-0.2]      [완료]
      ↓           ↓           ↓
   약함         이탈!        이탈!

견적전환:
  Home      →  Deal Hero  →  스크롤    →  하단CTA   →  Checkout
  [F=+2.0]    [F=-1.4]      [F=0]       [F=-0.2]     [F=0]
      ↓           ↓           ↓           ↓
   양호        이탈!        중립        이탈!
```
