## 1. 분석 프레임워크: 물리학 제1법칙

### 관성의 법칙

> "정지한 물체는 외부 힘이 작용하지 않으면 정지 상태를 유지한다"

고객의 기본 상태는 **"구매하지 않음"**이다. 전환시키려면:

```
F_net = F_motivation - F_friction > 0
```

| 요소 | 구성 |
|------|------|
| **F_motivation** (동기) | 가치 인식 + 신뢰 + 긴급성 + 개인화 |
| **F_friction** (마찰) | 불확실성 + 리스크 공포 + 선택 비용 + 인지 부하 |

### 핵심 원리

- **F_net > 0** → 전환 발생 (운동 시작)
- **F_net = 0** → 정체 (현상 유지)
- **F_net < 0** → 이탈 (반대 방향 운동)

## 3. 본질적 문제 진단

### 3.1 문제 1: 모바일 Hero에 CTA가 없다

```tsx
// PC에는 CTA가 있음 (line 195-203)
<button onClick={onApplyClick}>
  전화 연락 없는 견적 신청
</button>

// 모바일에는 CTA가 없음!
// 대신 이탈 버튼 2개만 있음 (line 117-138)
[기기 정보]  [다른 딜 비교]
```

**77%가 모바일 트래픽인데 Hero에 CTA가 없다.**

### 3.2 문제 2: 좋은 정보가 숨겨져 있다

| 정보 | 현재 위치 | 문제 |
|------|----------|------|
| 할인 금액 (70만원) | ConditionSection | Hero에 없음, 스크롤 필요 |
| 판매점 평점 | sr-only | 숨겨져 있음 |
| 판매점 리뷰 수 | StoreIntroSection | 페이지 맨 아래 |

```tsx
// StoreIntroSection.tsx - 평점이 sr-only로 숨겨져 있음!
{store.rating > 0 && (
  <div className="sr-only">  // ← 왜 숨김?!
    <meta itemProp="ratingValue" content={String(store.rating)} />
  </div>
)}
```

### 3.3 문제 3: 힘의 방향이 잘못됐다

```
현재 Hero의 힘 방향:
[기기 정보] → 상품 페이지로 이탈
[다른 딜 비교] → 스크롤해서 비교 (주의 분산)

있어야 할 힘 방향:
[견적 받기] → 전환
```

### 3.4 문제 4: 신뢰 정보의 단절

```
[Product 페이지]                    [Deal 페이지]

배터리 오래가요  카메라 좋아요        (아무것도 없음)
무게감이 있어요

전국 47개 매장 비교 완료             ✓ 판매점명 · 서울 강남

        ↓ 딜 클릭

        신뢰 정보가 사라진다!
```

**사용자 심리:**
- Product: "오 이 폰 평점 좋고, 배터리 오래간다네"
- Deal 클릭 후: "어? 아까 봤던 정보 어디 갔지? 이 판매점 믿어도 돼?"

### 4.2 화해의 6가지 원자 전략

#### 원자 #1: 숫자로 신뢰 구축

| 화해의 표현 | 본질 |
|------------|------|
| `4.73 (22,287)` | "2만 명이 검증했다" |
| `4점 이상 리뷰가 94%` | "대부분 만족했다" |
| `보습이 잘 돼요 137` | "137명이 같은 말을 했다" |

**원자 공식**: `[숫자] + [검증된 사실]`

#### 원자 #2: 질문하지 않고 답만 준다

| 고객의 숨은 질문 | 화해의 답변 방식 |
|-----------------|-----------------|
| "좋은 제품이야?" | `4점 이상 리뷰가 94%` |
| "뭐가 좋아?" | `좋았던 점: 보습이 잘 돼요` |
| "단점은?" | `아쉬운 점: 유분이 있어요` |
| "내 피부에 맞아?" | `건성 피부 만족도가 높은 제품` |
| "비싼 거야?" | `55% 27,900원` (원가 대비 표시) |

**원자 공식**: `[고객 질문 예측] → [즉시 답변]`

#### 원자 #3: 분류는 고객 언어로

```
좋았던 점           아쉬운 점
─────────────      ─────────────
보습이 잘 돼요 137   유분이 있어요 18
수분이 있어요 110   알러지 반응이 와요 7
흡수가 잘 돼요 101  담담해요 6
```

**관찰**:
- "보습력 우수" (X) → "보습이 잘 돼요" (O)
- "점도가 낮음" (X) → "끈적하지 않아요" (O)

**원자 공식**: `[고객이 실제로 하는 말] 그대로 사용`

#### 원자 #4: 판단은 고객 몫

화해는 **"이 제품이 좋다"고 말하지 않는다**.

```
성분 구성
● 1-2 낮은 위험  ● 3-6 중간 위험  ● 7-10 높은 위험
```

**원자 공식**: `[객관적 정보] + [판단 도구] → 고객이 스스로 결론`

#### 원자 #5: 개인화는 라벨링으로

```
건성 피부 만족도가 높은 제품

피부 타입별 성분
지성 피부 ████░░ 0개
건성 피부 ████████ 3개
민감성 피부 ██░░░░ 1개
```

**원자 공식**: `[고객 속성] ↔ [제품 속성] 매칭 시각화`

#### 원자 #6: 행동 유도는 맥락에 맞게

| 위치 | CTA | 이유 |
|-----|-----|------|
| 상단 | `장바구니 담기` `바로 구매` | 이미 결정한 고객용 |
| 리뷰 섹션 | `리뷰 305개 전체보기 >` | 더 확인하고 싶은 고객용 |
| 하단 고정 | `장바구니 담기` `바로 구매` | 스크롤 후 결정한 고객용 |

**원자 공식**: `[고객의 현재 마음 상태] → [그에 맞는 다음 행동]`

### 4.3 화해 전환 공식의 힘 합산

```
F_motivation (동기)
├─ 가격 앵커링: +60
├─ 시간 압박: +95
├─ 사회적 증거: +115
├─ 개인화: +40
├─ 랭킹 권위: +135
└─ 추가 혜택: +75
= 총 동기력: +520

F_friction (마찰)
├─ 리스크 제거: -90
├─ 정보 계층화: -75
├─ 성분 투명성: -90
├─ AI 분석: -95
├─ 듀얼 CTA: -30
├─ 하단 고정: -75
└─ 연관 추천: -30 (이탈 비용)
= 총 마찰력: -485

F_net = 520 - (-485) = +1005 → 강력한 전환 유도
```

## 5. 노피 vs 화해 Gap 분석

### 5.1 원자별 보유 현황

| 원자 ID | 화해 (전환 신호) | 노피 현재 (딜 페이지) | Gap |
|---------|-----------------|---------------------|-----|
| A01 | 평점 + 리뷰 수 | 부분: 후기 수는 하단, 평점은 숨김 | 신뢰 노출 약함 |
| A02 | 평점 분포(5점 비중) | 없음 | 확증 부재 |
| A03 | 사용자 리뷰 + 실사진 | 부분: 판매점 후기/이미지 있으나 하단 | 현실감 부족 |
| A04 | AI 리뷰 요약 | 없음 | 정보 피로 |
| A05 | 리뷰 토픽 + 카운트 | 없음 | 구체성 부족 |
| A06 | 위험도/조건 시각화 | 부분: 조건 정보는 있음, 시각화 약함 | 이해 부담 |
| A07 | 필터/탭 기반 자기검증 | 없음 | 자기검증 불가 |
| A08 | 적합성(피부타입 등) | 없음 | 적합성 불안 |
| A09 | 이미지 + 혜택 배지 | 부분: 이미지만 있음 | 혜택 직관화 부족 |

## 10. 핵심 원칙 요약

### 10.1 화해에서 배운 5가지 전환 원칙

| 원칙 | 잘못된 방식 | 올바른 방식 |
|------|-----------|-----------|
| **1. 숫자가 설득한다** | "좋아요" | "127명이 만족" |
| **2. 고객이 판단한다** | "추천합니다" | "94% 만족" |
| **3. 단점도 자산이다** | 숨김 | "알아두면 좋은 점" |
| **4. 시간이 압박한다** | 상시 | "오늘만 이 가격" |
| **5. 행동은 항상 가능하다** | 스크롤 필요 | 상시 노출 CTA |

### 10.2 화해의 3가지 소통 원칙

1. **숫자로 말한다**
   - 주관적 표현 배제
   - 검증된 데이터만 제시
   - 예: "좋아요" (X) → "94%가 만족" (O)

2. **고객 언어를 쓴다**
   - 전문 용어 배제
   - 고객이 실제로 하는 말 사용
   - 예: "보습력 우수" (X) → "보습이 잘 돼요" (O)

3. **판단을 강요하지 않는다**
   - 정보와 도구만 제공
   - 결론은 고객이 내린다
   - 예: "추천합니다" (X) → "건성 피부 만족도 높음" (O)

### 10.3 노피 적용 변환

**Before (노피):**
```
[큰 화면] [하루 종일 OK] [고성능] [사진 잘 나옴]
```

**After (화해 방식):**
```
이 폰을 산 고객 94%가 만족했어요

자주 언급된 장점
├─ 배터리 오래가요 (47회)
├─ 화면이 커요 (32회)
└─ 카메라 좋아요 (28회)

알아두면 좋은 점
├─ 무거워요 (12회)
└─ 발열 있어요 (8회)
```

### 10.4 해결 방향 한 문장

> **숨긴 것 꺼내고, CTA 넣고, 이탈 버튼 줄이고, Product와 Deal의 신뢰 정보를 연결한다.**

### 10.5 최소 구현 세트 (MVP 원자)

1. **A01**: 평점 + 리뷰 수 (상품 + 판매점)
2. **A10**: 할인 앵커링 (정가 대비 절감)
3. **A15**: 단일 CTA
4. **A13**: 안전/인증 문구
5. **A04/A05**: 리뷰 요약 + 키워드
6. **A14**: 랭킹/위치

### 10.6 첫 번째 실행 스텝

1. **CTA 단일화 + 안전 문구**부터 시작
2. 다음으로 **상품 신뢰 요약을 Hero로 승계**

이 두 가지만으로도 전환 시작점의 관성을 큰 폭으로 줄일 수 있다.

## 결론

### 본질적 문제 한 문장

> **노피의 본질적 문제: 좋은 데이터(평점, 할인액, 리뷰)를 갖고 있으면서 숨기고, 모바일 Hero에 CTA 없이 이탈 버튼만 있으며, Product에서 Deal로 넘어갈 때 신뢰 정보가 단절된다.**

### 물리학적 검증

```
Newton's First Law:
"물체는 외부 힘이 없으면 현재 상태를 유지한다"

현재: F_net < 0 → 이탈 (현재 상태: 미전환 유지)
개선: F_net > 0 → 전환 (새로운 상태: 견적신청)
```

### 화해가 하는 일은 단 하나:

> **"다른 고객들의 경험을 정리해서 보여준다"**

화해는 절대로:
- "이 제품 좋아요" 라고 말하지 않는다
- "당신에게 맞아요" 라고 말하지 않는다
- "사세요" 라고 말하지 않는다

화해는 오직:
- "2만 명이 평가했고, 94%가 만족했다"
- "137명이 '보습이 잘 된다'고 했다"
- "건성 피부 고객의 만족도가 높다"

**판단의 주체는 항상 고객이다.**
화해는 판단에 필요한 **정보와 도구**만 제공한다.

**고객이 스스로 "사야겠다"고 결론 내리게 한다.**

이것이 마찰을 최소화하면서 동기를 극대화하는 **물리학적으로 최적화된 전환 설계**다.
