## 1. 분석 프레임워크 요약

```
전환 발생 조건:
F_net = F_motivation - F_friction > 0

동기력 (F_motivation):
= (신뢰 × 가치 × 긴급성) / 인지부하

마찰력 (F_friction):
= 불확실성 + 리스크공포 + 선택비용 + 인지부하

엔트로피:
S = log(n)  // 선택지가 많을수록 S↑ → 이탈↑
```

- **노피_전환전략_물리학분석_v2.md**의 F_net 구조를 그대로 적용
- **화해_설득전략_물리학분석.md**의 “원자 단위 설득”을 당근 핵심 이미지에 매핑
- **노피_전체퍼널_원자분석_Home-Deal-Product.md**의 “신뢰 신호 단절”을 비교 기준으로 사용

## 3. 핵심 이미지별 물리학 분석

### 3.1 홈피드: 초기 운동량 생성
**이미지**: `당근_01_홈피드_상품목록.PNG`

- **관측**: 가격, 거리, 시간, 카테고리 칩이 리스트 카드에 동시 노출
- **물리학 해석**:
  - 거리/시간 = 위치에너지 안정화 → 불확실성 감소
  - 카테고리 칩 = 선택비용 분산 → 엔트로피 억제
- **효과**: F_motivation의 초기 질량(m) 확보

---

### 3.2 상세 상단: 충격량 극대화
**이미지**: `당근_17_상품상세_상단_이미지_가격_당근페이.PNG`

- **관측**: 대형 이미지 + 가격 + 당근페이 + 매너온도 동시 노출
- **물리학 해석**:
  - 0.3초 내 핵심 신호 집중 = **충격량 J = F × Δt 최대화**
  - 당근페이 아이콘 = 리스크공포 항 제거
  - 매너온도 = 신뢰 질량(m) 즉시 주입
- **효과**: F_motivation 급상승, F_friction 동시 하락

---

### 3.3 상세 하단: 불확정성 제거
**이미지**: `당근_18_상품상세_하단_설명_거래장소지도.PNG`

- **관측**: 상세 설명 + 거래 희망 장소 + 지도 + “100m 근처”
- **물리학 해석**:
  - 거래 장소 명시 = 위치 불확실성(Δx) 급감
  - 지도 시각화 = 심리적 거리 감소 → 신뢰 인력 강화
- **효과**: F_friction의 핵심 항목(불확실성) 제거

---

### 3.4 판매자 신뢰 확장: 신뢰 에너지 증폭
**이미지**:
- `당근_19_상품상세_판매자_다른물품목록.PNG`
- `당근_20_프로필_상단_매너온도_인증정보.PNG`
- `당근_21_프로필_판매물품_목록.PNG`
- `당근_22_프로필_받은후기_목록.PNG`

- **관측**: 판매물품 수, 인증 완료, 재거래 희망률/응답률, 후기 태그와 실제 리뷰
- **물리학 해석**:
  - 다수 판매물품 = “신뢰 질량” 확대
  - 응답률/응답속도 = 거래 성공 확률 가시화 → 불확정성 감소
  - 후기 태그 = 정보 압축 → 인지부하 감소 (화해 리뷰 요약 구조와 동일)
- **효과**: 신뢰 질량(m) 증가 → F_motivation 안정 유지

- **관측**: 온도 범위별 표정 변화 + 정의 문구
- **물리학 해석**:

  - 시각적 스케일링 = 인지부하 감소, 신뢰 해석 속도 상승
- **효과**: “신뢰 지표의 신뢰도” 자체를 강화

## 5. 에너지 흐름과 F_net 추정

### 5.1 상세 상단 단독 기준

```
F_motivation = (신뢰 1.2 × 가치 0.8 × 긴급성 0.3) / 인지부하 1.1 = 0.26
F_friction = 불확실성 0.6 + 리스크공포 0.4 + 선택비용 0.2 + 인지부하 0.3 = 1.5
F_net = -1.24  (상단만으로는 전환 불가)
```

### 5.2 프로필/후기 연계 후

```
신뢰 +0.9 (매너온도/인증/후기 태그)
불확실성 -0.4 (응답률/거래정보/지도)

F_friction = 1.1
F_net = -0.84  (여전히 낮으나 채팅 터널링 경로로 돌파 가능)
```

### 5.3 고관여(중고차) 케이스

```
비용 구조 분해 + 보험/시세 비교 적용 시
불확실성 -0.5, 인지부하 -0.2
F_friction = 0.4 ~ 0.6 수준

F_net ≈ 0 근처로 상승 → 채팅 CTA로 전환 발생
```

**결론**: 당근의 설득은 “정보 설득”이 아니라 **마찰 제거 + 채팅 터널링**의 결합이다.
