문제 정의: 왜 이 기능이 필요한가?

현재 상황

노피 에이전시 어드민은 판매점에게 단계별로 제한된 고객 데이터만 제공하고 있습니다. 판매점은 현재 진행 중인 고객(신청, 개통 진행중, 개통완료)만 확인할 수 있으며, 과거에 대응했거나 취소된 고객 데이터에는 접근할 수 없습니다.

핵심 문제점

1. DB 개당 판매 방식의 유산

  • 기존 비즈니스 모델: 판매점이 DB를 개당 구매하는 방식
  • 현재 시스템: 판매점이 "확인하기" 버튼을 클릭해야만 DB 열람 가능
  • 문제: 월구독제로 전환했지만, 여전히 과거 방식의 제약이 남아있음

2. 판매점의 제한된 고객 이력 접근

  • 과거 상담 이력을 확인할 수 없어 반복 고객 응대 어려움
  • 취소/반려된 고객 재접촉 불가능
  • 전체 고객 현황 파악 및 영업 전략 수립 곤란

3. 불필요한 프로세스 단계

  • 견적신청 → "확인하기" 클릭 → 개통 진행중
  • 판매점이 일일이 DB를 선별하는 비효율적인 과정

비즈니스 전략적 필요성

영업 전략으로서의 전체 데이터 공개

1. 판매점 역량 강화

  • 전체 고객 데이터 접근을 통한 자율적인 영업 활동 지원
  • 과거 상담 이력 기반 재접촉 및 후속 영업 가능
  • 취소/반려 고객에 대한 재어프로치 기회 제공

2. 월구독제 가치 제안 강화

  • "모든 고객 데이터를 무제한으로 확인하세요"
  • DB 개당 구매 방식 대비 명확한 차별점
  • 구독료 대비 높은 가치 제공

3. 경쟁 우위 확보

  • 타 플랫폼 대비 투명하고 개방적인 데이터 정책
  • 판매점의 자율성과 책임감 증대
  • 장기적인 파트너십 강화

설계 포인트

1. 버튼 기반 상태 변경

  • "개통완료"와 "대응완료" 두 개의 명확한 버튼을 세로 배치
  • 버튼 클릭 시 "김도현 고객을 '개통완료'로 변경하시겠습니까?" 형태의 확인 알럿 표시
  • 확인 후 즉시 DB 업데이트 및 화면 반영
  • 취소는 고객이 직접 하는 액션이므로 판매점 화면에서 제외

2. 테이블 형식 - 견적 정보 완전 표시

  • 11개 컬럼: 신청일시, 신청자, 모델, 통신사, 개통방식, 할인방식, 할부원금, 요금제, 상태, 액션, 메모
  • 모델명 + 저장용량을 2줄로 표시
  • 통신사는 배지로 시각화, 개통방식/할인방식/할부원금/요금제 금액은 텍스트로 표시
  • 할부원금과 요금제 금액은 font-weight: 600으로 강조하여 핵심 금액 정보 명확히 전달

3. 고객별 메모 관리

  • 개통 진행 중 특이사항을 실시간으로 기록
  • 메모 개수를 괄호 안에 표시하여 한눈에 파악 가능
  • 메모 버튼 클릭 시 모달 팝업으로 전체 메모 이력 조회 및 추가 작성

설계 포인트

1. 체크박스 필터 (탭 제거)

  • 여러 상태를 동시에 선택 가능 (예: 반려 + 취소 동시 조회)
  • 클릭 없이 한눈에 필터 상태 파악 가능
  • 탭 방식은 데이터를 숨기고 클릭을 강요하므로 제거
  • 실제 데이터 87%가 반려/취소이므로 다중 선택 필터 필수

2. 테이블 형식 - 견적 정보 완전 표시

  • 10개 컬럼: 완료일시, 고객명, 모델, 통신사, 개통방식, 할인방식, 할부원금, 요금제, 상태, 메모
  • 고객명 아래 연락처를 2줄로 표시 (견적 신청 페이지와 동일한 형식)
  • 모델명 + 저장용량을 2줄로 표시
  • 통신사는 배지로 시각화, 개통방식/할인방식/할부원금/요금제 금액은 텍스트로 표시
  • 할부원금과 요금제 금액은 font-weight: 600으로 강조하여 핵심 금액 정보 명확히 전달

3. 클릭 기반 상태 변경 UX

  • 상태 배지를 클릭하면 드롭다운 메뉴가 표시되어 직관적인 상태 변경 가능
  • 각 상태별로 변경 가능한 상태만 표시 (예: 대응완료 → 개통 진행중/개통완료)
  • 변경 전 확인 알럿으로 실수 방지
  • 취소 상태는 고객이 직접 취소한 것이므로 변경 불가 (배지 클릭 불가)

4. 통계 기반 영업 전략

  • 상단 통계 카드로 실제 전환율 11.5% 확인 (개통완료 47건 / 전체 410건)
  • 반려 2,860건 - 재제안 기회로 활용
  • 대응완료 101건 - 재접촉 우선순위 고객
  • 전체 데이터 공개를 통한 판매점 신뢰 강화
