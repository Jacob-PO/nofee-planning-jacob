## 📋 개요

노피(NOFEE) 서비스의 모든 데이터를 체계적으로 수집, 저장, 분석하는 중앙 데이터 저장소입니다.

**최종 업데이트**: 2025-11-20
**버전**: 3.7
**목적**: 데이터 기반 의사결정, 회사소개서 작성, 성과 분석

## 🔧 데이터 수집 방법

### 1. 환경 설정

```bash
# Python 의존성 설치
pip install -r scripts/config/requirements.txt

# 환경변수 설정
cp scripts/config/.env.example .env
# .env 파일 편집하여 DB 정보, GA4 정보 입력
```

### 2. 데이터 수집 실행

#### DB 데이터 수집
```bash
python3 scripts/collectors/collect_db_data.py
# → data/database/db_data_latest.json 생성
```

**수집 내용**:
- 핵심 비즈니스 지표 (가입자, 신청, 개통 등)
- 월별/일별 신청 추이
- 매장 퍼포먼스
- 실제 리뷰 데이터
- 전환율

#### GA4 데이터 수집
```bash
python3 scripts/collectors/collect_ga4_data.py
# → data/analytics/ga4_data_latest.json 생성
```

**수집 내용**:
- 전체 트래픽 지표 (최근 12개월)
- 디바이스 카테고리 분포
- 트래픽 채널 분석
- 월별 추이
- 상위 페이지
- 일평균/월평균 지표

#### 상품/매장 데이터 수집
```bash
python3 scripts/collectors/collect_product_store_data.py
# → data/products/product_store_data_latest.json 생성
```

**수집 내용**:
- 활성화된 상품 그룹
- 가격표 노출 매장
- 현재 판매중인 가격표
- 상품별/매장별/지역별 통계

### 3. 데이터 분석

#### 견적신청 퍼널 분석
```bash
python3 scripts/analyzers/analyze_application_funnel.py
# → reports/application_funnel_analysis.json 생성
```

**분석 내용**:
- 일반 견적 / 캠페인 견적 분석
- Step Code별 분포 (신청→견적→선택→예약→개통)
- 전체 퍼널 전환율
- 월별/일별 신청 추이
- 상품별 신청 통계

**상세 문서**: [docs/APPLICATION_FUNNEL_ANALYSIS.md](docs/APPLICATION_FUNNEL_ANALYSIS.md)

#### 서비스 버전별 개통율 분석
```bash
python3 scripts/analyzers/analyze_service_version.py
# → reports/service_version_analysis.json 생성
```

**분석 내용**:
- **버전 1** (동네성지 v1): 2025-07 ~ 2025-08 (일반견적 + 캠페인견적 통합)
- **버전 2** (동네성지 v2): 2025-09 ~ 2025-10 (검색 기능 추가, 어드민 개선)
- **버전 3** (공식 출시): 2025-11 ~ 현재 (베타 종료, 안정화)

## 🔄 B2B SaaS 생태계 전략

### 자율 경쟁 생태계의 탄생

**개통율 26배 향상이 만든 것**

검색 기능 추가로 고객이 몰리기 시작하자, 예상치 못한 일이 일어났습니다.
판매점들이 스스로 움직이기 시작했습니다.

**노피가 개입하지 않아도, 시장이 스스로 움직입니다**

매일 판매점들은:
- 전날 견적 신청 데이터를 확인합니다
- 경쟁 판매점의 가격을 비교합니다
- 자신의 가격표를 조정합니다

노피는 **데이터만 보여줄 뿐**입니다.
가격 조정, 재고 관리, 프로모션 결정 - 모두 판매점이 자율적으로 합니다.

### 판매점 어드민: B2B SaaS 모델

**초기 모델 (베타 기간: 2025년 7-10월)**
- 56개 판매점 직접 영업 (CEO 1:1 영업으로 모두 영입)
- 베타 기간 무료 제공
- 가치 검증: 개통율 26배 향상 증명

**유료 전환 (2025년 11월 1일~)**
- 베타 종료, 공식 서비스 출시
- **판매점 어드민 구독: 월 99,000원**
- **유료 전환율: 17.9%** (56개 중 10개 판매점 전환)

**현재 상태 (2025-11-20 기준)**
- **MRR (월간 반복 수익)**: 990,000원
  - 판매점 구독 수익: 10개 × 99,000원
  - 고객 매칭 수익: 별도 운영
- **이중 수익 구조**: B2B (판매점 구독) + B2C (고객 매칭)

### B2B SaaS 모델의 전략적 가치

**1. 네트워크 효과**
```
판매점 증가 → 가격 경쟁 심화 → 고객 증가 → 판매점 가치 증가
```
- 판매점 ↑ → 더 많은 상품, 더 낮은 가격
- 고객 ↑ → 판매점의 견적 신청 데이터 증가
- 데이터 ↑ → 판매점의 의사결정 정확도 향상
- 순환 구조가 플랫폼 가치를 기하급수적으로 증가

**2. 데이터 의존성 (높은 Lock-in)**
- 판매점은 견적 신청 데이터 없이 가격 조정 불가
- 매일 어드민 접속하여 데이터 확인 필수
- 경쟁사 대비 가격 경쟁력 확인 필수
- **이탈 비용이 매우 높음** (데이터 손실 = 경쟁력 손실)

**3. 확장성**
- **고객 플랫폼**: 마케팅비 투입 → 즉시 성장 (CAC 515원 검증)
- **판매점 SaaS**: CEO 직접 영업 → 안정적 MRR 확보
- **이중 수익 구조**: 양쪽 모두 성장 가능

**4. 시장 지배력**
- 판매점 56개 → 시장 커버리지 확보
- 유료 전환 10개 → PMF (Product-Market Fit) 검증
- 추가 영업으로 30개 목표 달성 시 MRR 300만원 돌파

### 로드맵: Retailer → Distributor 확장

**Phase 1: Retailer SaaS (현재)**
- MRR: 990,000원 (10개 Retailer)
- 유료 전환율: 17.9%
- 고객 플랫폼 개통율: 3.15% (안정적 운영)

**Phase 2: Retailer SaaS 확장 (2026년 Q1)**
- MRR: 3,000,000원 (30개 Retailer)
- 전환율: 53.6% (56개 중 30개)
- 추가 영업: 20개 판매점

**Phase 3: Distributor SaaS 출시 (2026년 Q2~)**

대한민국 휴대폰 유통 구조:
```
통신 3사 → Distributor (수백 개) → Retailer (수만 개) → 고객
```

Distributor = 시장의 중추:
- 가격·정책(리베이트)의 실질적 결정권자
- 1곳당 30~200개 Retailer 관리
- 전국 유통망의 핵심 허브

**확장 속도 100배:**
- Distributor 1곳 = Retailer 30~200곳
- Distributor 10곳 = Retailer 1,000~2,000곳
- MRR 수천만원 돌파

Distributor SaaS 핵심 기능:
- Retailer 가격·실적·재고 통합 대시보드
- 지역/기종/매장 단위 경쟁력 분석
- AI 기반 가격 정책 추천
- 정책 공지 → Retailer SaaS 자동 반영
- 실적 자동 보고, 캠페인 성과 분석

**Phase 4: 유통 OS (2027년~)**
- 가격·정책·재고·실적이 노피를 통해 흐름
- 통신 3사 공식 파트너십
- **대한민국 휴대폰 유통 표준 인프라**

**확장 전략**
1. **Retailer SaaS**: 기존 56개 중 30개 유료 전환
2. **Distributor SaaS 개발**: 2026년 Q1 출시 준비
3. **Distributor 영업**: 10곳 확보 = 1,000곳 효과
4. **유통 OS 구축**: 전국 유통망 데이터 통합

### 경쟁 우위

**vs 기존 중개 플랫폼**
- 기존: 중개수수료로 수익 (판매점에게 비용)
- 노피: SaaS 구독료 (판매점에게 마케팅 채널 제공)
- **판매점 입장**: 비용이 아닌 **투자**

**vs 전통 유통**
- 기존: 정보 비대칭, 불투명한 가격
- 노피: 완전한 투명성, 자율 경쟁
- **고객 입장**: 최저가 보장

**핵심 경쟁력**
- ✅ 검색 기능으로 검증된 26배 개통율 향상
- ✅ Retailer SaaS PMF 검증 완료 (유료 전환율 17.9%)
- ✅ 자율 경쟁 생태계 (노피 개입 최소화)
- ✅ 데이터 기반 의사결정 (판매점 일일 접속 유도)
- ✅ 이중 수익 구조 (B2B + B2C)
- ✅ 낮은 CAC (515원) + 안정적 MRR
- 🚀 **Distributor SaaS로 확장 속도 100배** (1곳 = 100곳 효과)
- 🚀 **유통망 전체를 관통하는 운영체제** (Retailer → Distributor → 통신사)
