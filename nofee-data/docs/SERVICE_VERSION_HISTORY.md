## 개요

노피는 고객과 판매점을 연결하는 휴대폰 개통 플랫폼으로, 서비스 모델을 3번의 큰 변화를 거쳐 진화해왔습니다. 각 버전은 고객/판매점의 VOC(Voice of Customer)를 반영하여 개통율과 사용자 경험을 지속적으로 개선했습니다.

### 서비스 버전 요약

| 버전 | 기간 | 주요 특징 | 개통율 |
|------|------|----------|--------|
| **버전 1: 일반견적** | 2025-07 ~ 2025-08 | 전국 평균 가격, 노피 중개 | **2.8%** |
| **버전 2: 캠페인 견적 (동네성지 v1)** | 2025-07 ~ 2025-08 | 판매점 직접 업로드 | **0.5%** |
| **버전 3: 동네 견적 (동네성지 v2)** | 2025-09 ~ 현재 | 검색 기능, 어드민 개선 | **6.0%** |

**핵심 성과**: 버전 3에서 버전 1/2 대비 **개통율 2배 이상 향상**

## 버전 1: 일반견적

### 📅 운영 기간
- **시작**: 2025년 7월
- **종료**: 2025년 8월 (버전 2와 병행 운영)
- **DB 테이블**: `tb_apply_phone`

### 🎯 서비스 모델

**핵심 컨셉**: 노피가 제공하는 전국 평균 가격으로 견적 제공 후, 고객 지역 기반 판매점 매칭

#### 프로세스 플로우
```
고객 신청 → 노피가 제공한 전국 평균 가격 확인
         → 고객 지역 정보 입력
         → 노피가 해당 지역 판매점에 DB 판매
         → 판매점이 고객에게 연락
         → 매장 방문 및 개통
```

### 📊 성과 데이터 (2025-07 ~ 2025-08)

#### 기본 지표
- **총 신청**: 1,666건
- **개통 완료**: 47건
- **개통 안함**: 3,878건 (반려/취소/대응완료)
- **진행중**: 176건
- **매장 구매 확정**: 1,020건

#### 전환율
| 전환 단계 | 전환율 | 비고 |
|----------|--------|------|
| **신청 → 개통** | **2.8%** | 35명 중 1명 개통 |
| 신청 → 매장구매 | 61.2% | 중간 단계 성과 양호 |
| 매장구매 → 개통 | 4.6% | 낮은 최종 전환 |

#### 월별 추이
| 월 | 신청 | 개통 | 개통율 |
|----|------|------|--------|
| 2025-07 | 241건 | 2건 | 0.8% |
| 2025-08 | 1,425건 | 6건 | 0.4% |

#### 개통 소요 시간
- **평균**: 15.4일
- **최소**: 0일
- **최대**: 31일

### 💡 주요 특징

**장점**:
- ✓ 노피가 전국 평균 가격을 제공하여 고객 신뢰도 확보
- ✓ 지역 기반 자동 매칭으로 고객 편의성 제공
- ✓ 매장 구매 전환율 61% (양호)

**단점**:
- ✗ 전국 평균 가격과 실제 판매점 가격 차이로 인한 불만
- ✗ 판매점 입장에서 수동적 DB 수신 (적극성 부족)
- ✗ 매장 구매 후 개통 전환율 낮음 (4.6%)
- ✗ 최종 개통율 2.8% (낮은 수준)

### 🔧 기술 스택
- **DB 테이블**: `tb_apply_phone`
- **주요 컬럼**:
  - `apply_product_group_code`: 노피가 제공한 상품
  - `apply_installment_principal`: 노피가 제시한 전국 평균 가격
  - `store_no`: 매칭된 판매점

## 버전 2: 캠페인 견적 (동네성지 v1)

### 📅 운영 기간
- **시작**: 2025년 7월
- **종료**: 2025년 8월 (버전 1과 병행, 버전 3 출시 전까지)
- **DB 테이블**: `tb_apply_campaign_phone`

### 🎯 서비스 모델

**핵심 컨셉**: "동네성지" 기능 도입 - 판매점이 직접 상품 및 가격 업로드, 고객이 원하는 상품 선택

#### 프로세스 플로우
```
판매점이 상품 등록 (가격 직접 설정)
         ↓
고객이 원하는 지역/상품 검색
         ↓
고객이 마음에 드는 판매점 상품 선택
         ↓
DB가 즉시 해당 판매점으로 전달
         ↓
판매점이 고객에게 연락
         ↓
매장 방문 및 개통
```

### 📊 성과 데이터 (2025-07 ~ 2025-08)

#### 기본 지표
- **총 신청**: 1,666건
- **개통 완료**: 8건
- **개통 안함**: 1,650건 (반려/취소/대응완료)
- **진행중**: 8건
- **매장 구매 확정**: 1,020건

#### 전환율
| 전환 단계 | 전환율 | 비고 |
|----------|--------|------|
| **신청 → 개통** | **0.5%** | 200명 중 1명 개통 (낮음!) |
| 신청 → 매장구매 | 61.2% | 버전 1과 동일 |
| 매장구매 → 개통 | 0.8% | 매우 낮은 전환 |

#### 월별 추이
| 월 | 신청 | 개통 | 개통율 |
|----|------|------|--------|
| 2025-07 | 241건 | 2건 | 0.8% |
| 2025-08 | 1,425건 | 6건 | 0.4% |

#### 개통 소요 시간
- **평균**: 3.0일
- **최소**: 1일
- **최대**: 9일

> **주목**: 개통 소요 시간은 버전 1(15.4일) 대비 **5배 빨라졌으나**, 개통율은 오히려 **떨어짐** (2.8% → 0.5%)

### 💡 주요 특징

**장점**:
- ✓ 판매점이 실제 판매 가능한 가격 직접 제시
- ✓ 고객이 원하는 판매점 직접 선택 가능
- ✓ DB 즉시 전달로 매칭 속도 빠름 (3일)
- ✓ "동네성지" 개념 도입으로 새로운 UX 제공

**단점**:
- ✗ 초기 버전으로 판매점 상품 업로드 미흡
- ✗ 검색 기능 부재로 고객이 원하는 상품 찾기 어려움
- ✗ 판매점 어드민 불편 (상품 등록 프로세스 복잡)
- ✗ 개통율 0.5%로 **버전 1보다도 낮음**
- ✗ 판매점/고객 간 커뮤니케이션 미흡

### 🔍 실패 원인 분석

1. **판매점 측 문제**:
   - 상품 등록 UI/UX 불편
   - 가격 업데이트 번거로움
   - 어드민 기능 부족

2. **고객 측 문제**:
   - 검색 기능 부재로 원하는 상품 찾기 어려움
   - 판매점별 가격 비교 불편
   - 신뢰할 수 있는 판매점 구분 어려움

3. **시스템 문제**:
   - 판매점-고객 매칭 품질 저하
   - 판매점 응대율 낮음
   - 고객 불만 대응 프로세스 미비

### 🔧 기술 스택
- **DB 테이블**: `tb_apply_campaign_phone`
- **주요 컬럼**:
  - `apply_product_group_code`: 고객이 선택한 판매점 상품
  - `store_no`: 고객이 직접 선택한 판매점

#### 주요 업데이트

1. **동네성지 검색 기능** ⭐
   - 지역/상품/가격대로 검색 가능
   - 판매점 비교 기능 추가
   - 실시간 가격 업데이트

2. **판매점 어드민 개선** ⭐
   - 상품 등록 프로세스 간소화
   - 일괄 가격 수정 기능
   - 판매 현황 대시보드
   - 고객 응대 관리 기능

3. **일반 견적 기능 제거** ⭐
   - 100% 동네성지로 통일
   - 판매점 주도형 비즈니스 모델 확립

4. **VOC 기반 UX 개선**
   - 판매점/고객 VOC 대규모 수집
   - 불편사항 개선
   - 고객 리뷰 시스템 강화

#### 프로세스 플로우
```

         ↓
가격/리뷰 비교 후 판매점 선택

DB 즉시 전달 + 판매점 알림

판매점 어드민에서 고객 정보 확인 및 응대
```

## 핵심 인사이트

### 1. 서비스 진화의 핵심은 "판매점 어드민"

**발견**: 버전 2의 실패는 판매점 어드민 부족 때문

- 버전 2: 판매점 어드민 미흡 → 상품 등록 저조 → 개통율 0.5%
- 버전 3: 판매점 어드민 개선 → 상품 활성화 → 개통율 6.0%

**교훈**: 양면 시장에서 공급자(판매점) 편의성이 최우선

### 2. 검색 기능은 필수

**발견**: 검색 기능 추가 후 개통율 12배 향상

- 버전 2: 검색 없음 → 고객이 상품 찾기 어려움
- 버전 3: 검색 추가 → 고객 편의성 향상 → 개통율 급상승

### 3. VOC 기반 개선이 효과적

**발견**: 판매점/고객 VOC 수집 및 반영 후 극적 개선

- 버전 3에서 대규모 VOC 수집
- 불편사항 개선 → 10월 개통율 15.3% 달성

### 4. 비즈니스 모델 통일의 중요성

**발견**: 일반 견적 + 캠페인 견적 병행 운영 시 혼란

- 버전 1/2 병행 운영: 고객 혼란, 개통율 낮음
- 버전 3에서 동네성지로 통일: 개통율 향상

### 5. 개통율 변동성 모니터링 필요

**발견**: 2025년 10월 15.3% → 11월 3.2%로 급락

**원인 가설**:
- 신청 건수 급감 (868 → 216건, -75%)
- 계절적 요인? 마케팅 축소?
- 판매점 활동 저하?

**대응 필요**:
- 원인 분석 (마케팅, 판매점 활동, 계절성)
- 신규 고객 유입 전략
- 판매점 활성화 캠페인
