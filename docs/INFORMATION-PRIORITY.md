# NOFEE 정보 우선순위 가이드

## 철학 기반 핵심 원칙

### Nofee가 고객에게 파는 것
> "후회 확률이 낮은 선택지에 대한 추천" (Information-based regret insurance)

고객이 알아야 할 것:
1. **무엇을** 사는가 (기기)
2. **얼마에** 사는가 (가격)
3. **믿을 수 있는가** (매장 신뢰도)
4. **어디서** 사는가 (위치)

나머지는 **부가 정보**다.

---

## 당근마켓 레퍼런스 분석

### 카드에 보이는 정보 (4개만)
```
┌─────────────────────────────────┐
│  [이미지]  │  상품명              │
│            │  위치 · 시간         │
│            │  가격 (굵게)         │
└─────────────────────────────────┘
```

**특징:**
- 이미지가 충분히 큼
- 상품명이 명확함
- 위치/시간은 회색 작은 텍스트
- 가격만 굵게
- **배지/태그 없음**
- **별점 없음** (카드에는)

## NOFEE 정보 우선순위

### 1단계: 상품 카드 (홈 리스트)

**목표:** 3초 안에 "이거 볼까?" 결정

| 우선순위 | 정보 | 표현 방식 |
|---------|------|----------|
| 1 | 기기 이미지 | 크고 깔끔하게 |
| 2 | 기기명 | 굵은 텍스트, 2줄 이내 |
| 3 | 매장명 + 정직도 | "모바일샵 · 정직 92%" |
| 4 | 위치 | 회색 작은 텍스트 "강남구" |
| 5 | 가격 | **가장 굵게, 맨 아래** |

**제외할 정보:**
- ~~통신사~~ (디테일에서)
- ~~할인유형~~ (디테일에서)
- ~~가입유형~~ (디테일에서)
- ~~요금제~~ (디테일에서)
- ~~옵션 개수 배지~~ (불필요)

**이유:**
카드는 "클릭할지 말지"만 결정하는 곳.
복잡한 조건은 클릭 후에 보여주면 됨.

---

### 2단계: 옵션 선택 (바텀시트)

**목표:** "어떤 조건으로 살까?" 비교 + **왜 가격이 다른지** 직관적 이해

**핵심 원칙:**
> 고객은 가격 차이의 **원인**을 알아야 한다.
> 조건(통신사, 가입유형, 할인방식, 요금제)과 가격의 연결고리를 보여줘야 한다.

| 우선순위 | 정보 | 표현 방식 | 이유 |
|---------|------|----------|------|
| 0 | 가격 차이 안내 | 회색 배경 텍스트 | 왜 가격이 다른지 설명 |
| 1 | 할부원금 | 가장 크게 (22px bold) | 핵심 비교 기준 |
| 2 | 최저가 표시 | 초록 배지 + 배경 | 최선의 선택 강조 |
| 3 | 월 총액 | 오른쪽 회색 | 실제 부담 |
| 4 | 핵심 조건 | 한 줄로 묶음 | 가격 결정 요인 |
| 5 | 요금제 | 라벨 + 값 | 가격에 영향 |

**가격 차이 안내 (바텀시트 상단):**
```
┌─────────────────────────────────────┐
│  통신사, 가입유형, 할인방식, 요금제에  │
│  따라 가격이 달라요                   │
└─────────────────────────────────────┘
```

**옵션 카드 레이아웃:**
```
┌─────────────────────────────────────┐  ← 초록 배경 (최저가)
│  15만원 [최저가]          월 8만원   │
│                                      │
│  [SKT] 번호이동 · 공시지원           │  ← 핵심 조건 한 줄
│  요금제 월 5만원 (5G 프리미어)        │  ← 라벨 + 값
├─────────────────────────────────────┤
│  20만원                   월 9만원   │
│                                      │
│  [KT] 기변 · 선택약정                │
│  요금제 월 7만원 (5G 슬림)            │
└─────────────────────────────────────┘
```

**조건 표시 규칙:**
- **1줄 (핵심 조건)**: `[통신사 배지] 가입유형 · 할인방식`
- **2줄 (요금제)**: `요금제 월 N만원 (요금제명)`

이렇게 하면 고객이 "왜 이 조건이 더 싼지" 직관적으로 이해할 수 있다.

---

### 3단계: 견적 확인 (상담 신청 전)

**목표:** "이 조건이 맞나?" 최종 확인 + 보호 안내

| 우선순위 | 정보 | 표현 방식 |
|---------|------|----------|
| 1 | 기기 + 조건 요약 | 상단 컴팩트하게 |
| 2 | 할부원금 | **가장 크게** |
| 3 | 가격 상세 | 월 할부 + 요금제 + 총액 |
| 4 | 매장 정보 | 이름 + 정직도 + 위치 |
| 5 | 보호 안내 | 노피 스냅샷 설명 |
| 6 | CTA | 상담 신청 버튼 |

## 핵심 차별화: "정직도"

당근은 개인간 거래라 신뢰 지표가 없음.
**Nofee의 핵심 차별점은 "정직도(TrustScore)"**

이걸 어떻게 보여줄까?

### 나쁜 예 (현재)
```
매장명 · ⭐ 4.5 · 정직 92%
```
→ 정보가 너무 많고 정직도가 묻힘

### 좋은 예 (개선)
```
매장명
정직 92% · 강남구
```
→ 매장명 바로 아래에 정직도 강조
→ 별점은 상세에서만

---

## 정보 표현 규칙

### 가격
- 모든 가격은 `n만원` 형식
- 예: 15만원, -14만원, 0만원

### 정직도
- `정직 92%` 형식
- 색상으로 등급 표현 (초록/파랑/주황/빨강)

### 위치
- 시군구만 표시: "강남구"
- 거리는 있으면 표시: "1.2km"

### 통신사
- 컬러 배지로 구분
- SKT: 빨강, KT: 검정, LGU+: 핑크

## 신청내역 페이지

### 고객이 신청내역을 보는 이유

고객이 신청내역 페이지에 오는 순간, 머릿속에 있는 질문들:

1. **"내가 뭘 신청했더라?"** → 기기 확인
2. **"어디에 신청했지?"** → 매장 확인
3. **"언제 신청했더라?"** → 날짜 확인 (연락 안 오면 "며칠 됐지?" 판단)
4. **"지금 어떤 상태야?"** → 상담 대기인지, 진행중인지
5. **"얼마였지?"** → 가격 확인

이 다섯 가지가 **3초 안에** 눈에 들어와야 한다.

---

### 4단계: 신청내역 카드 (리스트)

**목표:** "내가 뭘, 어디에, 언제 신청했고, 지금 어떤 상태인지" 즉시 파악

| 우선순위 | 정보 | 표현 방식 | 고객의 질문 |
|---------|------|----------|------------|
| 1 | 기기 이미지 | 100x100 정사각형 | "이게 그거야" (시각적 즉시 인식) |
| 2 | 기기명 | 굵은 텍스트, 2줄 | "뭘 신청했더라" |
| 3 | **매장 + 날짜** | 한 줄로 묶음 | "어디에, 언제 신청했지" |
| 4 | 가격 | **가장 굵게** | "얼마였지" |
| 5 | **상태** | 컬러 배지, 오른쪽 | "지금 어떤 단계야" |

**날짜가 중요한 이유:**
- "3일 전에 신청했는데 아직 연락 없네" → 진행 상황 판단
- "이거 최근 거야 예전 거야" → 여러 신청 내역 구분
- "언제 했더라" → 기본적인 타임라인 파악

**제외할 정보 (카드에서):**
- ~~통신사~~ → 클릭하면 상세에서
- ~~할인유형~~ → 클릭하면 상세에서
- ~~가입유형~~ → 클릭하면 상세에서
- ~~요금제~~ → 클릭하면 상세에서
- ~~정직도~~ → 이미 신청했으니 불필요

**레이아웃:**
```
┌─────────────────────────────────────┐
│ [이미지]  │  아이폰 16 Pro 256GB    │
│           │  모바일샵 · 오늘        │  ← 매장+날짜 한 줄
│           │                         │
│           │  15만원      [상담 대기] │  ← 가격+상태
└─────────────────────────────────────┘
```

**상태 배지:**
- `상담 대기` - 파란색 (#131FA0) - 매장 연락 기다리는 중
- `상담중` - 주황색 (#FF6F0F) - 매장과 대화 진행 중
- `후기 작성` - 노란색 - 개통 완료, 후기 남겨주세요
- `거래 완료` - 초록색 - 후기까지 완료
- `취소됨` - 회색

---

### 5단계: 신청 상세 (바텀시트)

**목표:** "내가 어떤 조건으로 신청했는지" 상세 확인

고객이 상세를 클릭하는 이유:
- "정확히 어떤 조건이었지?"
- "요금제가 뭐였더라?"
- "유지기간이 몇 개월이지?"

| 우선순위 | 정보 | 표현 방식 |
|---------|------|----------|
| 1 | 기기 + 통신사/가입유형 | 상단 요약 |
| 2 | **약속된 가격** | 가장 크게 (가격 breakdown) |
| 3 | 약속된 조건 | 그리드 (할인유형, 요금제, 유지기간, 부가서비스) |
| 4 | 매장 정보 | 이름, 지역, 정직도 |
| 5 | 거래 후기 | 후기 작성했으면 표시 |
| 6 | 액션 버튼 | 상태에 따라 다름 |

**약속된 가격 표시:**
```
약속된 가격
────────────────────────────────────
출고가                    1,430,000원
공시지원금               -500,000원
매장할인                 -780,000원
────────────────────────────────────
할부원금                    15만원
```

**약속된 조건 표시:**
```
┌──────────────┬──────────────┐
│ 할인유형      │ 요금제        │
│ 공시지원      │ 5만원        │
├──────────────┼──────────────┤
│ 최소유지      │ 부가서비스    │
│ 6개월        │ 2개          │
└──────────────┴──────────────┘
```

---

### 6단계: 후기 작성

**목표:** "실제로 이대로 됐어?" 확인받기

고객에게 물어볼 핵심 질문:
> **"실제로 이대로 진행됐나요?"**

이게 전부다. 복잡하게 만들지 마라.

| 우선순위 | 정보 | 표현 방식 |
|---------|------|----------|
| 1 | 약속된 조건 요약 | 상단에 다시 보여줌 ("이 조건 기억나세요?") |
| 2 | **일치 여부** | 큰 버튼 2개 |
| 3 | 뭐가 달랐나요? | 체크리스트 (불일치 시만) |
| 4 | 평점 | 별 5개 |
| 5 | 한 줄 후기 | 텍스트 (선택) |

**일치 여부 버튼:**
```
┌─────────────┐  ┌─────────────┐
│     ✓       │  │     !       │
│  네, 그대로요  │  │  아니요, 달랐어요 │
└─────────────┘  └─────────────┘
```

**달랐다면 뭐가 달랐나:**
- 가격이 달랐어요
- 요금제가 달랐어요
- 부가서비스가 추가됐어요
- 유지기간이 달랐어요
- 기타 조건이 달랐어요

---

### 신청내역에서 "정직도"의 역할

**홈에서:** 정직도는 "이 매장 믿어도 될까?" 판단 기준
**신청내역에서:** 이미 선택한 매장이라 크게 의미 없음

그래서:
- 카드에서는 정직도 안 보여줌 (이미 선택함)
- 상세에서만 보여줌 (참고용)

**후기 작성 후:** 내 후기가 매장 정직도에 반영됨
→ "솔직한 후기가 다른 사람에게 도움이 된다" 느낌

## 원칙

> **"덜 보여주는 게 더 좋다"**
>
> 사용자는 정보를 읽지 않는다. 스캔한다.
> 3초 안에 핵심이 눈에 안 들어오면 이탈한다.
>
> 카드에서 모든 걸 보여주려 하지 마라.
> 카드는 "클릭하게 만드는 것"이 목표다.
> 상세 정보는 클릭 후에 보여주면 된다.
