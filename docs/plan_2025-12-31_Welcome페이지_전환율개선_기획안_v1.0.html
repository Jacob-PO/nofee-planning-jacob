현재 문제점

1. "왜 로그인해야 하지?" - 가치 명제 부재

기존 Welcome 페이지는 단순히 "카카오로 로그인" 버튼만 있었습니다. 사용자 입장에서는:

  • "로그인하면 뭐가 좋은데?"
  • "굳이 개인정보 줘야 해?"
  • "그냥 가격만 보고 싶은데..."

로그인의 혜택이 명확하지 않아 이탈이 발생합니다.

2. 이전 맥락과 단절됨

사용자가 Deal 페이지에서 특정 상품(예: 아이폰 16 Pro, 월 55,000원)을 보고 왔는데, Welcome 페이지에서는 그 상품 정보가 전혀 보이지 않습니다.

  • "내가 보던 상품은 어디 갔지?"
  • "다시 찾아야 하나?"
  • 맥락 단절로 인한 불안감 → 이탈
3. 체류시간 26.8초 - 결정 장애

Welcome 페이지의 평균 체류시간이 26.8초로 전체 페이지 중 가장 짧습니다.

  • 너무 짧은 시간 = 바로 이탈
  • 결정할 정보가 없어서 빠르게 나감
  • 혜택/이유가 명확하면 체류시간 증가 예상

개선 방향

핵심 전략
1. 신규가입 혜택(1만원 쿠폰)을 명확히 강조
2. Deal 페이지에서 본 상품 정보를 그대로 보여주기
3. 할인된 가격을 직접 보여줘서 전환 동기 부여

Deal 페이지에서 "견적 신청" 클릭 시, 해당 상품 정보를 sessionStorage에 저장합니다. Welcome 페이지에서 이 정보를 불러와 사용자가 보던 상품과 할인된 가격을 보여줍니다.

3. 할인 가격 시각화

할부원금에서 1만원을 뺀 가격을 직접 보여줍니다.

  • 기존 가격: 취소선 처리 (예: 990,000원)
  • 할인 가격: 강조 표시 (예: 980,000원)
  • "1만원 추가할인 받기" 문구로 행동 유도

  • 긴급성 없음 - "오늘 가입 시 우선 배정" 같은 문구 부재
  • 신뢰 지표 없음 - "★4.8 평점" 같은 문구 부재
2. 약관 동의 - 엔트로피 높음
  • 5개 체크박스 = 5번 의사결정 = 높은 엔트로피
  • 진행률 표시 없음 (전체 몇 단계 중인지 모름)
  • "전체 동의" 버튼은 있음 (긍정적)
3. 희망정보 - 스킵 불균형
  • Step 1 (지역)은 스킵 불가
  • Step 2 (통신사/가격대/제조사)만 X 버튼으로 스킵 가능
  • 왜 이 정보가 필요한지 명확한 가치 제안 없음
  • 진행률 표시 없음

회원가입 플로우 개선 제안

제안: 약관 동의 전에 먼저 회원가입을 완료하고, 이후 단계에서 약관/추가정보 수집

이유: 현재 약관 동의 단계(F_net = -0.5)가 가장 큰 이탈 구간. 먼저 가입시키고 나중에 정보를 받으면 이탈 감소

현재 플로우

1. 카카오 로그인
2. 약관 동의 (5개) ← 이탈!
3. POST /sign-up/kakao (회원가입)
4. 희망지역
5. 희망정보
6. 완료

개선 플로우

1. 카카오 로그인
2. POST /sign-up/kakao (회원가입) ← 먼저!
3. 약관 동의 (정보 업데이트)
4. 희망지역 (스킵 가능)
5. 희망정보 (스킵 가능)
6. 완료

백엔드 수정 필요
현재 POST /login/sign-up/kakao API는 필수 약관 3개가 모두 "Y"여야만 회원 생성 가능
프론트엔드만으로는 플로우 변경 불가능 → 백엔드 팀과 협의 필요

예상 작업량: 백엔드 30분~1시간, 프론트엔드 30분~1시간
