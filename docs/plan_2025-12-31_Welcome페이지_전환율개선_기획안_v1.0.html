현재 문제점

1. "왜 로그인해야 하지?" - 가치 명제 부재

기존 Welcome 페이지는 단순히 "카카오로 로그인" 버튼만 있었습니다. 사용자 입장에서는:

  • "로그인하면 뭐가 좋은데?"
  • "굳이 개인정보 줘야 해?"
  • "그냥 가격만 보고 싶은데..."

로그인의 혜택이 명확하지 않아 이탈이 발생합니다.

2. 이전 맥락과 단절됨

사용자가 Deal 페이지에서 특정 상품(예: 아이폰 16 Pro, 월 55,000원)을 보고 왔는데, Welcome 페이지에서는 그 상품 정보가 전혀 보이지 않습니다.

  • "내가 보던 상품은 어디 갔지?"
  • "다시 찾아야 하나?"
  • 맥락 단절로 인한 불안감 → 이탈
3. 체류시간 26.8초 - 결정 장애

Welcome 페이지의 평균 체류시간이 26.8초로 전체 페이지 중 가장 짧습니다.

  • 너무 짧은 시간 = 바로 이탈
  • 결정할 정보가 없어서 빠르게 나감
  • 혜택/이유가 명확하면 체류시간 증가 예상

개선 방향

핵심 전략
1. 신규가입 혜택(1만원 쿠폰)을 명확히 강조
2. Deal 페이지에서 본 상품 정보를 그대로 보여주기
3. 할인된 가격을 직접 보여줘서 전환 동기 부여
1. 신규가입 혜택 명시

2. 상품 정보 연계 (Deal → Welcome)

변경 후

사용자가 보던 상품 이미지
상품명, 통신사, 용량
할부원금: 990,000원 → 980,000원

Deal 페이지에서 "견적 신청" 클릭 시, 해당 상품 정보를 sessionStorage에 저장합니다. Welcome 페이지에서 이 정보를 불러와 사용자가 보던 상품과 할인된 가격을 보여줍니다.

3. 할인 가격 시각화

할부원금에서 1만원을 뺀 가격을 직접 보여줍니다.

  • 기존 가격: 취소선 처리 (예: 990,000원)
  • 할인 가격: 강조 표시 (예: 980,000원)
  • "1만원 추가할인 받기" 문구로 행동 유도

기술 구현 사항

Deal → Welcome 데이터 전달

  • sessionStorage에 displayInfo 저장
  • 저장 항목: productGroupNm, installmentPrincipal, monthPrice, storage, carrier, productGroupThumbImage
  • Welcome 페이지에서 useEffect로 불러와 표시
수정된 파일
  • DealDetailClient.tsx: displayInfo를 pendingEstimate에 추가 저장
  • welcome/page.tsx: 상품 정보 표시 UI, 할인 가격 계산, PC 레이아웃 수정
