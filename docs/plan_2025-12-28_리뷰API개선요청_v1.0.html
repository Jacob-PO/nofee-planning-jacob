리뷰 API 개선 요청

리뷰 데이터 조회 시 발생하는 문제 해결을 위한 백엔드 수정 요청

작성일: 2025.12.28 작성자: 제이콥
DB 현황
tb_review_store_phone (47건) + tb_review_store_phone_virtual (295건) = 총 342건

요청 1: 리뷰 통계 데이터 추가 1순위

문제: 리뷰가 10건만 조회되어 프론트에서 만족도 계산이 부정확함

API: POST /review
필요한 데이터
- 전체 리뷰 중 4점 이상 리뷰 수
-- 쿼리 참고 SELECT COUNT(*) as total, SUM(CASE WHEN rating >= 4 THEN 1 ELSE 0 END) as high_rating_count FROM ( SELECT rating FROM tb_review_store_phone WHERE deleted_yn = 'N' AND product_group_code = #{productGroupCode} UNION ALL SELECT rating FROM tb_review_store_phone_virtual WHERE product_group_code = #{productGroupCode} ) combined

요청 2: 판매점 목록에 대표 리뷰 포함 2순위

문제: 판매점별 리뷰 조회 시 빈 결과 반환되어 추가 API 호출 필요

API: POST /store/list
필요한 데이터
- 각 판매점의 대표 리뷰 1건 (내용, 평점)
-- 쿼리 참고 (판매점별 대표 리뷰 1건) SELECT content, rating, user_nm FROM ( SELECT content, rating, user_nm, created_at FROM tb_review_store_phone WHERE deleted_yn = 'N' AND store_no = #{storeNo} UNION ALL SELECT content, rating, user_nm, created_at FROM tb_review_store_phone_virtual WHERE store_no = #{storeNo} ) combined ORDER BY rating DESC, created_at DESC LIMIT 1

요청 3: 리뷰 조회 개수 확대 3순위

문제: Length 파라미터가 10으로 제한됨

API: POST /review
필요한 변경
- Length 파라미터 최대값을 50으로 확대
